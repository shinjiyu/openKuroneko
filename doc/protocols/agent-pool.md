# Agent 共享池协议（Agent Pool）

**版本**：v1.1  
**日期**：2026-03-05

---

## 1. 概述

以 **agent**（外脑）为主体：外脑技能池泛化存储所有内脑历史产出；内脑生命周期内技能与知识仅存在于内脑空间，**仅在实例结束时**才归档至外脑技能库。新实例创建时**按目标做一轮相关技能选择**后注入，而非全量注入；内脑通过**「查询可用技能」接口/工具**在需要时主动拉取（当前实现为查外脑技能库，未来可外挂不同技能库）。

参与方：

- **生产者**：内脑实例（在 `workDir/.brain/skills/` 等产出技能与知识）
- **消费者**：外脑池子（launch 时按目标选择相关技能注入；实例退出时将实例技能与知识归档回外脑池）

---

## 2. 存储形态

### 2.1 外脑技能池目录（单池）

```
<obDir>/agent-pool/
└── .brain/
    ├── skills.md          # 技能索引（与 workDir/.brain/skills.md 同格式）
    └── skills/
        └── <category>/
            └── <id>.md    # 技能文件
```

- `obDir`：外脑工作目录。同一外脑下所有任务共享此唯一池。

### 2.2 技能索引格式（skills.md）

与内脑一致：TSV，表头 `# skills index: id\tcategory\ttitle\ttags\tts`，数据行 `id\tcategory\ttitle\ttags\tts`。

---

## 3. 语义约定

### 3.1 创建内脑时：相关技能选择后注入

- **时机**：外脑池子 `launch(goal, ...)` 时，在创建 `workDir`、写入 `goal.md` 与 `status` 之后，启动内脑进程之前。
- **行为**：根据当前目标 `goal` 从外脑技能池中**做一轮相关性选择**（如关键词/标题/标签匹配），选出 top-K 条相关技能，**仅将选中的技能**写入 `workDir/.brain/skills.md` 与 `workDir/.brain/skills/`。
- **效果**：新实例只携带与本任务相关的技能子集，避免泛化池全量注入。

### 3.2 内脑运行时：查询可用技能（接口形态）

- **形态**：内脑提供「查询可用技能」的**接口**（如 `SkillProvider.search(query, topK)`），以便未来外挂不同技能库（外脑池、远程 API 等）。当前实现为该接口查询外脑技能库（由外脑在 spawn 时通过环境变量传入池路径）。
- **使用**：当 LLM 认为当前工具不足以解决问题时，**先发起一轮查询**（调用对应工具），再根据返回的技能内容决策。工具实现调用上述接口，返回匹配技能的摘要或全文供 LLM 使用。

### 3.3 内脑生命周期内：技能与知识仅在内脑空间

- 归因阶段生成的技能在内脑生命周期内只存在于 `workDir/.brain/skills/`，不实时同步到外脑。
- 其它知识/经验同理，仅在内脑空间中积累。

### 3.4 内脑结束时：归档至外脑技能库

- **时机**：内脑实例退出时（RUNNING → EXITED），由外脑在 `onInstanceExit` 中触发。
- **行为**：将 `workDir/.brain/skills.md` 与 `workDir/.brain/skills/**/*.md` 合并到 `<obDir>/agent-pool/.brain/`；可选扩展：其它知识/经验一并归档。
- **合并规则**：按技能 `id`；池中无则添加，有则仅当实例 `ts` 不早于池中时覆盖。

---

## 4. 错误与边界

| 情况 | 约定 |
|------|------|
| 外脑池不存在或为空 | 创建时跳过注入；归档时创建目录并写入。 |
| 选择结果为空 | 仍创建实例，其 `.brain/skills` 为空，可依赖运行时「查询可用技能」工具。 |
| 未配置技能库路径（如独立运行内脑） | 查询工具返回「未配置外部技能库」等提示。 |
| 多实例并发退出 | 合并顺序不确定；后完成的覆盖先完成的，可接受。 |

---

## 5. 与现有模块的关系

- **InnerBrainPool**：launch 时调用「相关技能选择 + 注入」；onInstanceExit 时由外脑调用「归档」。
- **内脑 CLI**：通过环境变量 `OPENKURONEKO_OB_SKILL_POOL` 接收外脑池路径（由池子 spawn 时注入）；据此创建 SkillProvider 并注册「查询可用技能」工具。
- **BrainFs**：内脑本地技能读写、检索逻辑不变；与池子共用同一索引与文件格式。
- **技能渐进式披露**：内脑对本地技能采用「先索引、后按需 get_skill_content」的渐进式披露，详见 [技能渐进式披露设计](../designs/skills-progressive-disclosure.md)。
