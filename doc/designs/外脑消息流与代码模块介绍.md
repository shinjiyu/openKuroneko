# 外脑消息流与代码模块介绍

本文档梳理外脑（Outer Brain）的**消息流**与**代码模块**，便于通读该部分代码。

---

## 1. 消息流总览

```
                    ┌─────────────────────────────────────────────────────────┐
                    │                    外脑进程 (obDir)                       │
  IM / 用户          │                                                         │
      │              │  ┌──────────────┐    onMessage     ┌─────────────────┐  │
      ▼              │  │ ChannelAdapter│ ──────────────► │ index.ts         │  │
  ┌─────────┐       │  │ (feishu/dingtalk/ │              │ onInboundMessage│  │
  │ 飞书/钉钉 │       │  │  cli/webchat)     │              └────────┬────────┘  │
  │ WebSocket│       │  └──────────────┘    start(onMessage)        │           │
  └────┬────┘       │         ▲                    │                │           │
       │            │         │                    │                ▼           │
       │            │  ┌───────┴───────┐     ┌─────┴─────┐   ┌──────────────┐   │
       │            │  │ChannelRegistry │     │ ThreadStore │   │ DM? 直接      │   │
       │            │  │ startAll()     │     │ UserStore   │   │ Group @? 直接  │   │
       │            │  └────────────────┘     └────────────┘   │ Group 其他? 队列 │   │
       │            │                                          └───────┬──────┘   │
       │            │                                                  │           │
       │            │              ┌──────────────────────────────────┼───────┐   │
       │            │              ▼                                  ▼       │   │
       │            │     ┌─────────────────┐              ┌─────────────────┐   │
       │            │     │ ConversationLoop │              │ ParticipationEngine│   │
       │            │     │ process(msg)     │              │ shouldSpeak()    │   │
       │            │     │ → LLM + 工具      │              │ → SPEAK → 进 Loop │   │
       │            │     │ → reply_to_user  │              └─────────────────┘   │
       │            │     └────────┬────────┘                                       │
       │            │              │                                                │
       │            │              ▼                                                │
       │            │     ┌─────────────────┐     send()      ┌─────────────────┐   │
       │            │     │ ChannelRegistry │ ──────────────► │ ChannelAdapter  │───┼──► IM
       │            │     │ send(msg)        │                 │ .send(outbound) │   │
       │            │     └─────────────────┘                 └─────────────────┘   │
       │            └─────────────────────────────────────────────────────────────┘
       │
       │  另路：中转 relay 广播
       │  ┌──────────────────────────────────────────────────────────────────────┐
       │  │ CLI: relayIngestRef.current(threadId, userId, content, ts, opts)      │
       │  │  → threadStore.ensureThread + appendUser                               │
       │  │  → ob.onRelayMessageIngested() → 同上的参与决策 / 直接 ConversationLoop │
       │  └──────────────────────────────────────────────────────────────────────┘
```

- **入站**：ChannelAdapter 收到 IM 消息后调用 `onMessage(msg)`，即 `index.ts` 的 `onInboundMessage`。
- **DM**：直接 `runConversation(msg)`，进对话环。
- **群 @ 了本机**：先写 thread，再 `runConversation(msg, { skipAppendUser: true })`。
- **群未 @**：进延迟队列，到期后 `runParticipateDecision` → `ParticipationEngine.shouldSpeak`，若 SPEAK 再 `runConversation`。
- **出站**：ConversationLoop 里通过工具 `reply_to_user` 调用 `channelRegistry.send(outbound)`，由 Registry 根据 `thread_id` 选 Channel 发回 IM。

## 2. 模块与文件对照

### 2.1 入口与组装

| 模块 | 路径 | 职责 |
|------|------|------|
| **CLI 入口** | `src/cli/outer-brain.ts` | 解析参数、创建 LLM、注册 Feishu/钉钉等 adapter、创建 createOuterBrain、注入 relayIngestRef、启动外脑。 |
| **外脑工厂** | `src/outer-brain/index.ts` | createOuterBrain()：创建 ThreadStore / UserStore / ChannelRegistry / SoulLoader / ConversationLoop / ParticipationEngine / BlockEscalationManager / PushLoop，实现 onInboundMessage、runConversation、runParticipateDecision、onRelayMessageIngested，并在 start() 里 channelRegistry.startAll(onInboundMessage)。 |

通读顺序建议：先看 `index.ts` 顶部注释和 `onInboundMessage`、`runConversation`、`runParticipateDecision`、`onRelayMessageIngested` 的分支逻辑，再看 `createOuterBrain` 里各子模块的构造与注入。

---

### 2.2 频道层（入站 → 外脑、外脑 → 出站）

| 模块 | 路径 | 职责 |
|------|------|------|
| **类型定义** | `src/channels/types.ts` | InboundMessage、OutboundMessage、MessageAttachment、ChannelAdapter 接口（start(onMessage)、send(msg)、stop）、Thread、GroupInfo。 |
| **注册表** | `src/channels/registry.ts` | ChannelRegistry：register(adapter)、startAll(onMessage)（对每个 adapter 调用 adapter.start(onMessage)）、send(msg)（按 thread_id 解析 channel_id，调对应 adapter.send）。 |
| **飞书适配器** | `src/channels/adapters/feishu.ts` | 飞书 Bot：start() 内建 WebSocket 长连接/事件分发，收到消息后组 InboundMessage 调 onMessage(msg)；send() 发文本/post（@ 标签）、附件，并可选转发到 relay。 |
| **钉钉适配器** | `src/channels/adapters/dingtalk.ts` | 钉钉 Stream 模式，同上 start/onMessage/send。 |
| **CLI 适配器** | `src/channels/adapters/cli.ts` | 本地命令行，读 stdin 组 msg 调 onMessage。 |
| **WebChat 适配器** | `src/channels/adapters/webchat.ts` | Web 聊天 UI，SSE/HTTP，消息进 onMessage。 |

通读顺序：types.ts → registry.ts → 选一个 adapter（如 feishu.ts）看 start 如何调 onMessage、send 如何发回。

---

### 2.3 存储（线程与用户）

| 模块 | 路径 | 职责 |
|------|------|------|
| **线程存储** | `src/threads/store.ts` | ThreadStore：getOrCreate(msg)/ensureThread(threadId, channelId, ts)、appendUser/appendAssistant、getHistory(threadId)、持久化到 \<obDir\>/threads/*.meta.json 与 *.history.jsonl。 |
| **用户存储** | `src/users/store.ts` | UserStore：用户注册、channel 绑定、resolveUser、updateLastSeen，供外脑/内脑解析展示名与推送目标。 |

消息流里：每条入站消息都会在需要时 getOrCreate/ensureThread 并 appendUser；ConversationLoop 用 getHistory 拼上下文。

---

### 2.4 对话环（LLM 与回复）

| 模块 | 路径 | 职责 |
|------|------|------|
| **对话环** | `src/outer-brain/conversation-loop.ts` | ConversationLoop.process(msg, soul, opts)：写 thread 历史（若未 skip）、拉 soul + 内脑状态 + 知识库（若有）+ thread 历史 + 当前消息拼 messages，调 LLM，处理工具调用（含 reply_to_user）；最终把回复 appendAssistant 并 channelRegistry.send。buildMessages 里拼 DM/群的不同上下文。 |
| **回复工具** | `src/outer-brain/tools/reply-to-user.ts` | reply_to_user 工具：参数 thread_id、content、reply_to_msg_id，内部 channelRegistry.send()，供 LLM 选 thread 发回复。 |

通读顺序：conversation-loop.ts 的 process() 主流程 → buildMessages/buildSystemPrompt → 工具调用分支 → reply-to-user 的 call()。

---

### 2.5 群聊参与决策

| 模块 | 路径 | 职责 |
|------|------|------|
| **参与决策** | `src/outer-brain/participation.ts` | ParticipationEngine.shouldSpeak(msg, recentGroupMessages, soul, innerStatus)：规则预筛（level、冷却、5min 上限、level1 非问句、最短长度等），通过则用 LLM 做 SPEAK/SILENT 分类；返回 true 表示应发言，由 index 调 runConversation。 |

index 里群消息（非 @）会进延迟队列，到期后取最近群历史拼成 recentGroupMessages 再调 shouldSpeak。

---

### 2.6 BLOCK 升级与推送

| 模块 | 路径 | 职责 |
|------|------|------|
| **BLOCK 升级** | `src/outer-brain/block-escalation.ts` | BlockEscalationManager：内脑 BLOCK 时按 channel 优先级发通知，等待 target_user 回复（escalation_wait_ms）；收到回复后发解封 directive。onInboundMessage 时检查是否是某次 BLOCK 的解除回复。 |
| **推送环** | `src/outer-brain/push-loop.ts` | PushLoop：轮询各内脑实例 output 文件，解析 BLOCK/COMPLETE/PROGRESS；BLOCK 交 BlockEscalationManager；COMPLETE 给 goal_origin_user 发完成通知。 |

这两块是「内脑 → 外脑 → 用户」的主动推送路径，与「用户 → 外脑 → 对话环」的入站路径并列。

---

### 2.7 Soul 与工具集

| 模块 | 路径 | 职责 |
|------|------|------|
| **Soul 配置** | `src/outer-brain/soul.ts` | SoulLoader：读 soul.md，解析 name、system_prompt、participation 等，供 ConversationLoop 与 ParticipationEngine 使用。 |
| **工具汇总** | `src/outer-brain/tools/index.ts` | 导出 createReplyToUserTool、createSearchThreadTool、createSendFileTool、set_goal、stop_inner_brain、send_directive、read_inner_status、list_inner_brains 等，在 index 里注入 ConversationLoop。 |

---

## 3. 推荐通读顺序

1. **类型与入口**  
   `src/channels/types.ts`（InboundMessage / OutboundMessage / ChannelAdapter）→ `src/outer-brain/index.ts`（消息流分支与 createOuterBrain 组装）。

2. **入站到对话**  
   `src/channels/registry.ts`（startAll、send）→ 选一个 `src/channels/adapters/feishu.ts`（start、onMessage、send）→ `src/outer-brain/conversation-loop.ts`（process、buildMessages、工具调用、reply_to_user）。

3. **存储**  
   `src/threads/store.ts`（getOrCreate、appendUser、getHistory）→ `src/users/store.ts`（register、resolveUser）。

4. **群聊决策**  
   `src/outer-brain/participation.ts`（shouldSpeak 规则 + LLM 分类）。

5. **BLOCK 与推送**  
   `src/outer-brain/block-escalation.ts`（waitForResolution、onInboundMessage）→ `src/outer-brain/push-loop.ts`（轮询 output、BLOCK/COMPLETE 处理）。

6. **CLI 与 relay**  
   `src/cli/outer-brain.ts`（adapter 注册、relayIngestRef 注入、onRelayMessageIngested 的调用时机）。
