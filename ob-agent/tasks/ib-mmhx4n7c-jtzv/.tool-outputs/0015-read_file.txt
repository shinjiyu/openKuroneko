# 飞书消息接入调研报告

**调研时间**：2026-03-08  
**调研目标**：了解飞书开放平台的接入流程、权限要求，以及单聊、群聊、图片、语音消息的API实现方式

---

## 一、飞书开放平台接入流程

### 1.1 基本流程

飞书开放平台的应用开发流程主要包括以下步骤：

1. **注册开发者账号**
   - 访问飞书开放平台：https://open.feishu.cn
   - 使用飞书App扫描二维码登录
   - 进入开发者后台

2. **创建应用**
   - 在开发者后台创建企业自建应用
   - 填写应用基本信息（名称、图标、描述等）
   - 选择应用类型（企业自建应用/商店应用）

3. **配置应用能力**
   - 添加"机器人"能力（必需，用于消息收发）
   - 配置应用主页、事件订阅等（可选）

4. **申请权限**
   - 在"权限管理"中申请所需的API权限（scope）
   - 权限需要企业管理员审核通过

5. **开发与测试**
   - 获取 App ID 和 App Secret
   - 使用测试应用进行API调用测试
   - 在开发者后台的API调试台进行接口测试

6. **发布应用**
   - 配置可用范围（哪些部门/用户可以使用）
   - 提交版本发布
   - 企业管理员审核通过后生效

### 1.2 关键配置项

- **App ID**：应用唯一标识
- **App Secret**：应用密钥，用于获取 access_token
- **机器人能力**：必须开启，用于消息收发
- **事件订阅**：配置回调地址，接收消息和事件推送
- **权限管理**：申请所需的API权限

## 二、权限要求

### 2.1 权限机制

飞书开放平台采用 **scope（权限范围）** 机制管理API调用权限：

- 出于安全考虑，应用需要申请相应的权限
- 权限需要企业管理员审核通过
- 应用的API权限决定了应用能使用哪些开放能力

### 2.2 消息相关权限

发送和接收消息需要申请以下权限：

#### 发送消息权限
- `im:message` - 获取与发送单聊、群聊消息
- `im:message:send_as_bot` - 以应用身份发消息

#### 接收消息权限
- `im:message.p2p_msg` - 获取用户发给机器人的单聊消息
- `im:message.group_msg` - 获取群组中@机器人的消息
- `im:message.group_msg:readonly` - 读取群聊消息（历史权限）

#### 文件相关权限
- `im:resource` - 获取与上传图片、文件资源

### 2.3 权限配置流程

1. 进入应用的"权限管理"页面
2. 搜索或筛选需要的权限
3. 点击"申请权限"
4. 填写申请理由
5. 等待企业管理员审核

## 三、消息API实现方式

### 3.1 API调用基础

#### 获取 access_token

所有API调用需要先获取 access_token：

```http
POST https://open.feishu.cn/open-apis/auth/v3/tenant_access_token/internal/
Content-Type: application/json

{
  "app_id": "cli_xxxxxxxxxx",
  "app_secret": "xxxxxxxxxxxxxxxxxx"
}
```

返回：
```json
{
  "tenant_access_token": "t-xxxxxxxxxx",
  "expire": 7200
}
```

#### API调用规范

- **基础URL**：`https://open.feishu.cn/open-apis/`
- **认证方式**：在HTTP Header中添加 `Authorization: Bearer {access_token}`
- **内容类型**：`Content-Type: application/json`

---

### 3.2 单聊消息

#### 发送单聊消息

**API接口**：`POST /im/v1/messages?receive_id_type=open_id`

**请求示例**：
```json
{
  "receive_id": "ou_xxxxxxxxxx",  // 用户的 open_id
  "msg_type": "text",
  "content": "{\"text\":\"你好，这是测试消息\"}"
}
```

**关键参数**：
- `receive_id_type`：接收者ID类型
  - `open_id`：用户的开放平台ID（推荐）
  - `user_id`：用户的企业内部ID
  - `union_id`：用户的统一ID
- `receive_id`：接收者的ID值
- `msg_type`：消息类型（text/post/image/file/audio/media/interactive等）
- `content`：消息内容（JSON字符串）

**前提条件**：
- 应用必须开启机器人能力
- 用户必须在机器人的可用范围内
- 需要申请消息发送权限

---

### 3.3 群聊消息

#### 发送群聊消息

**API接口**：`POST /im/v1/messages?receive_id_type=chat_id`

**请求示例**：
```json
{
  "receive_id": "oc_xxxxxxxxxx",  // 群聊的 chat_id
  "msg_type": "text",
  "content": "{\"text\":\"大家好，这是群消息\"}"
}
```

**关键参数**：
- `receive_id_type`：使用 `chat_id`
- `receive_id`：群聊的 chat_id（以 `oc_` 开头）

**前提条件**：
- 机器人必须在该群聊中
- 机器人在群内拥有发言权限
- 需要申请群聊消息发送权限

#### 获取群聊列表

**API接口**：`GET /im/v1/chats`

可以获取机器人所在的群聊列表。

---

### 3.4 图片消息

#### 步骤1：上传图片

**API接口**：`POST /im/v1/images`

**请求格式**：`multipart/form-data`

**关键参数**：
- `image`：图片文件（二进制）
- `image_type`：图片类型
  - `message`：用于发送消息
  - `avatar`：用于头像

**支持的图片格式**：
- JPEG
- PNG
- WEBP
- GIF
- TIFF
- BMP
- ICO

**返回示例**：
```json
{
  "code": 0,
  "data": {
    "image_key": "img_v2_xxxxxxxxxx"
  }
}
```

#### 步骤2：发送图片消息

**API接口**：`POST /im/v1/messages`

**请求示例**：
```json
{
  "receive_id": "ou_xxxxxxxxxx",
  "msg_type": "image",
  "content": "{\"image_key\":\"img_v2_xxxxxxxxxx\"}"
}
```

**注意事项**：
- 必须先上传图片获取 `image_key`
- `image_key` 有时效性，建议获取后立即使用
- 上传图片时 `image_type` 必须选择 `message`

---

### 3.5 语音/音频消息

#### 步骤1：上传音频文件

**API接口**：`POST /im/v1/files`

**请求格式**：`multipart/form-data`

**关键参数**：
- `file`：音频文件（二进制）
- `file_type`：文件类型，使用 `stream` 表示音频流
- `file_name`：文件名（包含扩展名）

**支持的音频格式**：
- opus（推荐格式，飞书原生支持）
- mp3
- wav
- aac
- m4a

**返回示例**：
```json
{
  "code": 0,
  "data": {
    "file_key": "file_v2_xxxxxxxxxx"
  }
}
```

#### 步骤2：发送音频消息

**API接口**：`POST /im/v1/messages`

**请求示例**：
```json
{
  "receive_id": "ou_xxxxxxxxxx",
  "msg_type": "audio",
  "content": "{\"file_key\":\"file_v2_xxxxxxxxxx\",\"duration\":3000}"
}
```

**关键参数**：
- `file_key`：上传音频后获得的文件标识
- `duration`：音频时长（毫秒）

**注意事项**：
- 飞书语音消息最大时长：30分钟
- 推荐使用 opus 格式（音质好、文件小）
- 建议文本长度不超过500字（避免转换超时）

## 四、消息类型总览

飞书支持以下消息类型：

| 消息类型 | msg_type | 说明 |
|---------|----------|------|
| 文本消息 | `text` | 普通文本消息 |
| 富文本消息 | `post` | 支持富文本格式，可包含文字、@、图片、链接等 |
| 图片消息 | `image` | 图片消息，需先上传获取 image_key |
| 文件消息 | `file` | 文件消息，需先上传获取 file_key |
| 音频消息 | `audio` | 语音/音频消息，需先上传获取 file_key |
| 视频消息 | `media` | 视频消息，需先上传获取 file_key |
| 消息卡片 | `interactive` | 交互式卡片消息，支持按钮、表单等 |
| 分享群名片 | `share_chat` | 分享群聊名片 |
| 分享个人名片 | `share_user` | 分享用户名片 |
| 表情消息 | `sticker` | 表情包消息 |

## 五、频率限制

### 5.1 消息发送限频

为避免消息发送频繁对用户造成打扰，飞书对消息发送有以下限制：

- **单聊消息**：向同一用户发送消息有频率限制
- **群聊消息**：向同一群组发送消息有频率限制
- **API调用频率**：具体接口有不同的QPS限制

### 5.2 文件上传限制

- **图片上传**：支持常见图片格式，文件大小有上限
- **文件上传**：文件大小不得超过20MB，不可上传空文件
- **上传频率**：5 QPS，10000次/天

## 六、事件订阅

### 6.1 接收消息事件

要接收用户发送给机器人的消息，需要配置事件订阅：

1. **配置事件订阅地址**
   - 在应用后台配置事件订阅的请求地址（URL）
   - 该地址需要能够接收HTTP POST请求

2. **订阅消息事件**
   - 添加事件订阅：`im.message.receive_v1`
   - 当用户给机器人发送消息时，飞书会推送事件到配置的地址

3. **事件推送格式**
```json
{
  "schema": "2.0",
  "header": {
    "event_id": "xxxxxxxxxx",
    "event_type": "im.message.receive_v1",
    "create_time": "1609459200000"
  },
  "event": {
    "sender": {
      "sender_id": {
        "open_id": "ou_xxxxxxxxxx"
      }
    },
    "message": {
      "message_id": "om_xxxxxxxxxx",
      "message_type": "text",
      "content": "{\"text\":\"你好\"}"
    }
  }
}
```

## 七、开发建议

### 7.1 认证与安全

- 妥善保管 App Secret，不要泄露
- access_token 有时效性（2小时），需要缓存并定期刷新
- 使用 HTTPS 协议进行API调用

### 7.2 消息发送

- 单聊优先使用 `open_id` 作为用户标识
- 群聊使用 `chat_id` 作为群组标识
- 发送图片、音频等媒体消息，需要先上传获取资源key

### 7.3 错误处理

- API返回的 `code` 字段为 0 表示成功
- 非零值表示失败，需要根据错误码进行相应处理
- 常见错误：
  - 权限不足：检查是否申请了相应权限
  - 用户不在可用范围：检查应用的可用范围配置
  - 频率限制：降低调用频率，稍后重试

### 7.4 测试与调试

- 使用开发者后台的API调试台进行测试
- 先在测试环境验证功能，再发布到生产环境
- 注意查看API返回的错误信息，便于定位问题
