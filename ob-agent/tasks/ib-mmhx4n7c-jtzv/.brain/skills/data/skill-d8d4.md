# 多文档对比分析报告生成模式

> category: data | id: skill-d8d4 | 2026-03-08T15:49:27.724Z

场景：需要对比两个或多个主题（如平台、技术方案、产品）的能力、特性、限制等维度，并输出结构化的对比分析报告。

步骤：
1. 读取所有待对比主题的详细文档/数据
2. 确定对比维度（接入复杂度、功能完整性、关键限制、适用场景等）
3. 为每个维度建立对比表格，明确标注优势方（用⭐等符号）
4. 总结核心差异点和选择建议
5. 提供技术实现建议（如适用）
6. 输出完整的Markdown格式报告，包含评分表和建议

验证：
- 报告覆盖所有要求的对比维度
- 每个维度都有明确的对比结论
- 包含可操作的选择建议
- 报告长度适中（5000-6000字符），结构清晰
