# 技术方案多源调研与整合模式

> category: web | id: skill-0a74 | 2026-03-08T15:38:58.640Z

场景：需要对某项技术或方案进行系统性调研，收集多种实现路径、开源项目、协议文档等资料，并输出结构化的调研报告

步骤：
  1. 使用 web_search 以不同关键词组合进行多轮搜索（如"XX API 文档"、"XX 开源 项目"、"XX 协议 实现"等）
  2. 对关键搜索结果使用 fetch 操作，深入获取项目主页、文档、API 说明等详细信息
  3. 交叉对比不同方案的优缺点、适用场景、技术栈、成熟度等
  4. 将调研结果整理为结构化文档，包含：概述、核心方案对比、技术细节、实施建议、参考资源等章节
  5. 额外创建资源链接汇总文档，便于后续快速查阅

验证：生成完整的调研报告文档（至少包含 3 个以上方案的对比分析），并附带可用的参考链接列表
