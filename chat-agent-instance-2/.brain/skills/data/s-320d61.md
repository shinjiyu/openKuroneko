# 目标用户身份未确认或平台访问受限，需要在数据严重不足情况下生成分析报告

> category: data | id: s-320d61 | 2026-03-06T09:15:23.765Z

场景：目标用户身份未确认或平台访问受限，需要在数据严重不足情况下生成分析报告

步骤：
1. 整合所有前期调查数据（本地扫描、平台搜索、跨平台发现）
2. 构建结构化报告框架，包含：基础信息表、跨平台推测、兴趣特征、社交网络、信息来源清单、可靠性分级
3. 对每个信息项明确标注：可靠性等级（⭐⭐⭐高/⭐⭐中/⭐低/❌无数据）、信息来源、备注说明
4. 量化数据完整性（用百分比和进度条展示各维度覆盖率）
5. 单独设立"分析盲区"章节，详细说明平台限制、身份确认障碍、数据缺口
6. 提供"补充数据需求"清单，按优先级（P0/P1/P2）列出缺失信息及获取途径
7. 给出多条后续路径建议（终止/等待输入/替代方案），明确各方案的条件和预期效果

验证：
- 报告中所有推测性信息均标注了可靠性等级
- 数据完整性评估有量化指标（如"综合完整性：4%"）
- "分析盲区"章节清晰说明了限制原因和影响范围
- 提供了可操作的后续行动建议
