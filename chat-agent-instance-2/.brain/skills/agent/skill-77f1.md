# 系统环境自检与能力盘点标准流程

> category: agent | id: skill-77f1 | 2026-03-07T15:29:56.030Z

场景：Agent 启动时或进入新环境时，需要全面了解可用工具集、权限边界和运行环境特性

步骤：
  1. 基础环境探测：执行 `pwd && whoami && uname -a` 获取工作目录、用户身份和操作系统信息
  2. 运行时环境验证：检查 Node.js (`node --version`)、npm (`npm --version`) 等关键工具的版本
  3. 工具可用性测试：逐个验证核心工具（如 `which playwright`，或通过实际调用测试）
  4. 权限边界探测：尝试访问预期外的路径（如父目录），记录安全违规信息以明确沙箱边界
  5. 资源限制检查：执行 `ulimit -a` 和 `df -h` 了解系统资源限制和磁盘空间
  6. 网络能力验证：通过 web_search 执行简单搜索，确认互联网访问能力
  7. 历史痕迹分析：读取 overview.md、project-structure.md 等现有文档，了解项目背景和历史任务
  8. 状态文件读取：读取 .brain/controller-state.json 等框架管理文件，了解当前运行模式
  9. 汇总生成报告：将所有发现整理成结构化的 Markdown 报告，包含工具清单、环境特性、权限边界、能力总结

验证：
  - 成功生成包含工具清单、环境特性、权限边界的完整报告
  - 至少发现 1 个权限限制或能力边界
  - 报告文件字数 > 2000 字符
