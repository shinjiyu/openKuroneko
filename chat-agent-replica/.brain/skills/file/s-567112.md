场景：里程碑状态验证与同步（交付物已存在的情况）

步骤：
1. 读取 knowledge.md 历史记录，确认交付物是否已在之前周期生成
2. 读取里程碑状态文件（milestones.md），确认当前状态
3. 读取交付物文件，验证存在性和内容完整性
4. 调用 edit_file 更新里程碑状态（[Active] → [Completed]）
5. 调用 read_file 二次验证状态更新成功

验证：通过 read_file 确认 milestones.md 中目标里程碑状态为 [Completed]，且交付物文件可正常读取且内容符合预期
