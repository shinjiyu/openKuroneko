# 需要分析极简项目的关键文件（配置文件、入口脚本、文档等），确认里程碑完成度

> category: file | id: s-111966 | 2026-03-06T09:15:23.741Z

场景：需要分析极简项目的关键文件（配置文件、入口脚本、文档等），确认里程碑完成度

步骤：
  1. 读取 .brain/knowledge.md 了解已掌握的事实，识别已分析过的文件
  2. 读取 .brain/goal.md 和 milestones.md 明确当前目标和里程碑状态
  3. 读取 .brain/constraints.md 了解执行约束
  4. 读取已有的项目文档（如 overview.md, project-structure.md）
  5. 识别项目中的关键文件类型：配置文件、入口脚本、核心业务文件、文档
  6. 对于已记录在 knowledge.md 中的文件，跳过重复分析；对于未记录的文件，执行读取
  7. 综合所有信息，确认"已识别并读取所有关键文件"

验证：
  - 所有存在的文件类型都已识别（配置、入口、业务、文档）
  - 核心业务文件的内容已掌握（通过 knowledge.md 或直接读取）
  - 能够明确回答"项目的关键文件有哪些"
