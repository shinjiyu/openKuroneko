# 需要将多个里程碑/执行周期收集的碎片化信息整理成结构化概览文档

> category: web | id: s-0e290c | 2026-03-06T09:15:23.764Z

场景：需要将多个里程碑/执行周期收集的碎片化信息整理成结构化概览文档

步骤：
1. 读取 knowledge.md 获取完整历史上下文（所有已记录的事实）
2. 识别与当前任务相关的历史记录条目（按时间戳或关键词筛选）
3. 设计分类框架（建议章节：数据源状态、搜索结果、信息分类、可行性评估、建议、总结）
4. 明确区分三类信息：可确认事实（有证据支撑）、高概率推测（基于逻辑推理）、不确定因素（需要验证）
5. 生成结构化 Markdown 文档（使用表格、列表、分级标题提升可读性）
6. 更新里程碑状态为 [Completed]
7. 二次验证文档存在性和里程碑状态

验证：
- 文档包含所有关键信息分类章节
- 区分了事实/推测/不确定因素三类信息
- 提供了明确的下一步建议或结论
- 里程碑状态成功更新为 [Completed]
