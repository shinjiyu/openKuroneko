# 需要验证或完成某个里程碑（如目录扫描、文件读取、报告生成等），但 knowledge.md 中可能已有历史完成记录

> category: file | id: s-627174 | 2026-03-06T09:15:23.749Z

场景：需要验证或完成某个里程碑（如目录扫描、文件读取、报告生成等），但 knowledge.md 中可能已有历史完成记录

步骤：
1. 读取 knowledge.md，检查是否有相关历史事实记录（文件存在性、内容完整性、里程碑完成状态）
2. 如历史记录完整，执行最小化验证（如 find 扫描目录、stat 检查文件存在性），避免重复读取文件内容
3. 使用 edit_file 显式同步 milestones.md 状态（[Active] → [Completed]），即使 knowledge.md 已有记录
4. 二次读取 milestones.md 验证状态更新成功，形成闭环

验证：
- milestones.md 中目标里程碑状态已变为 [Completed]
- knowledge.md 中新增本次完成的事实记录
- 零冗余调用（未重复读取已知文件内容）
