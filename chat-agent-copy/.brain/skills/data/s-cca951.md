# 多源信息整合与结构化汇总方法

> category: data | id: s-cca951 | 2026-03-06T10:03:00.752Z

场景：需要将多个来源的报告文件整合为一个结构化的汇总文档，按维度分类呈现

步骤：
1. 读取所有相关报告文件（knowledge.md、user-profile、final-report 等）
2. 识别关键信息维度（身份背景、活跃状态、社交关系、内容主题等）
3. 为每个维度创建子分类（如身份背景 → 微博核心身份、跨平台关联、真实身份推测）
4. 对每条信息标注可靠性等级（⭐⭐⭐ 100%、⭐⭐⭐ 高、⭐⭐ 中等）
5. 区分已确认事实、高概率推测、待确认信息
6. 添加信息质量评估章节，说明各维度完整度
7. 生成建议与下一步章节

验证：
- 汇总文档包含所有关键维度
- 信息来源清晰标注
- 可靠性分级明确
- 文档结构清晰、可读性强
- 里程碑状态正确更新
