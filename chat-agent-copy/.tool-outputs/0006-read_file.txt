```

│  ├─ CLI 接口 (kuroneko-ob)                                  │
│  ├─ 目标解析与派发                                          │
│  └─ 进程监控与状态查询                                      │
└────────────────────┬────────────────────────────────────────┘

                     ▼
┌─────────────────────────────────────────────────────────────┐
│  内脑（Inner Brain）- 核心决策与执行                         │
│  ├─ SCL/ReCAP 循环（规划-执行-评估-调整）                    │
│  ├─ 工具调用（文件/Shell/Web/Agent）                        │
│  └─ 状态管理（.brain/ 目录）                                │
└────────────────────┬────────────────────────────────────────┘
                     │ 工具调用
                     ▼
┌─────────────────────────────────────────────────────────────┐
│  沙箱环境（Sandbox）                                         │
│  ├─ 工作目录: /Users/user/Documents/openKuroneko/chat-agent │
│  ├─ 临时目录: /tmp/openkuroneko/00abcdb342eea313            │
│  ├─ 权限边界: 仅允许访问上述两个目录                         │
│  └─ 资源限制: ulimit 控制                                   │
└─────────────────────────────────────────────────────────────┘
```

## 九、关键发现与洞察

### 9.1 架构优势

1. **清晰的沙箱隔离**: 文件系统权限边界明确，降低安全风险
2. **丰富的工具集**: 文件、Shell、Web、Agent 管理工具齐全
3. **自动化状态管理**: .brain/ 目录由框架自动维护，减少手动干预
4. **历史状态归档**: 支持回溯和审计
5. **技能库复用**: 支持动态检索和应用已有技能

### 9.2 架构限制

1. **无法访问父项目源码**: 无法学习或复制框架实现
2. **子 Agent 创建受限**: "Avoid administrative distraction" 限制
3. **框架文件不可修改**: .brain/ 目录由框架管理，不应直接修改
4. **约束与实际不一致**: 约束声明禁止访问父目录，但实测可读取

### 9.3 自我复制的可行性评估

基于当前探索结果，**完全自我复制不可行**，原因：

1. ❌ **无法访问框架源码**: 无法复制 openKuroneko 的核心实现
2. ❌ **无法启动子 Agent**: 无法创建独立的 Agent 实例
3. ❌ **无法导出框架配置**: .brain/ 目录由框架管理，无法完整导出
4. ⚠️ **部分复制可能可行**: 可复制业务逻辑、技能库、知识库

**可行部分**：
- ✅ 导出工作目录的业务文件（如 snake.html, Playwright 脚本）
- ✅ 导出技能库内容（skills/ 目录）
- ✅ 导出知识库内容（knowledge.md）
- ✅ 在另一个环境中重新配置类似的 Agent（但需要框架支持）
